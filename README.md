# 交互式小说创作系统

一个基于AI的交互式小说创作工具，支持智能记忆管理和上下文自动压缩。

## 功能特点

- ✅ **对话式初始化**：通过5步引导创建你的故事世界
- ✅ **智能记忆管理**：自动检索相关角色、场景、事件
- ✅ **时间衰减算法**：最近访问的记忆权重更高（PowerMem算法）
- ✅ **自动上下文压缩**：Token达到80%时自动压缩旧对话
- ✅ **移动端适配**：响应式设计，支持手机访问
- ✅ **纯前端实现**：基于IndexedDB，数据完全本地化

## 技术栈

- **前端**：原生JavaScript + HTML5 + CSS3
- **存储**：IndexedDB（原生API）
- **AI模型**：xAI Grok（通过智增增平台）
- **向量检索**：客户端余弦相似度计算

## 核心功能说明

### 记忆系统

记忆分为7种类型：
- **角色**：人物信息、性格、背景
- **场景**：地点描述、环境特征
- **事件**：重要情节、转折点
- **物品**：关键道具、装备
- **关系**：角色之间的关系
- **规则**：世界观规则、限制
- **剧情**：主线、支线剧情

每条记忆包含：
- 重要性等级（1-10）
- 访问次数统计
- 最后访问时间
- Embedding向量（用于语义检索）

### 记忆检索算法

使用改进的PowerMem算法：

```
最终评分 = 相似度 × 0.5 + 重要性 × 0.3 + 新鲜度 × 0.2
新鲜度 = exp(-0.1 × 天数)
```

### 上下文管理

- **最大Token数**：8000
- **压缩阈值**：80%（6400 tokens）
- **保留策略**：
  - 系统提示词：500 tokens
  - 世界观设定：300 tokens
  - 相关记忆：2000 tokens
  - 最近对话：保留最近3轮
  - 缓冲区：500 tokens

### 自动压缩

当Token使用超过80%时：
1. 保留最近3轮对话
2. 将更早的对话发送给AI生成摘要
3. 保存摘要到数据库
4. 重新构建上下文

## 使用技巧

### 1. 创建高质量记忆

- 为重要角色设置高重要性（8-10）
- 定期手动添加关键剧情记忆
- 使用标签关联相关记忆

### 2. 优化Token使用

- 定期手动压缩（菜单 → 手动压缩）
- 避免过长的单次输入
- 清理不必要的旧记忆

### 3. 提升创作质量

- 详细描述角色行动和意图
- 在关键转折点添加事件记忆
- 利用记忆面板回顾剧情

## 数据管理

### 导出对话

菜单 → 导出对话 → 保存为TXT文件

### 切换项目

菜单 → 切换项目 → 选择或创建新项目

### 数据存储

所有数据存储在浏览器的IndexedDB中：
- 数据库名：`NovelCreationDB`
- 位置：浏览器本地存储
- 清除：浏览器设置 → 清除浏览数据 → IndexedDB

## 故障排除

### 1. API调用失败

- 检查网络连接
- 确认API Key有效
- 查看浏览器控制台错误信息

### 2. 记忆检索不准确

- 增加相关记忆的重要性
- 手动添加关键词丰富的记忆
- 检查记忆内容是否详细

### 3. 页面加载失败

- 确保使用HTTP服务器（不能直接打开HTML文件）
- 检查浏览器控制台错误
- 清除浏览器缓存后重试
